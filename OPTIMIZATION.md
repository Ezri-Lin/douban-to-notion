# 性能优化说明

本次优化主要针对可维护性和性能，包括以下几个方面：

## 1. 并发处理优化

### 封面验证并行化
- **文件**: `cover_validator.py`
- **优化**: 将串行的封面验证改为并行处理
- **效果**: 使用线程池同时验证多个URL，大幅提升验证速度
- **配置**: 通过 `MAX_WORKERS` 环境变量控制并发数（默认10）

### 封面获取并行化
- **文件**: `douban.py`
- **优化**: 将串行尝试多个封面源改为并行尝试
- **效果**: 同时尝试Amazon、Goodreads、GoogleBooks等源，取第一个成功的结果
- **配置**: 通过 `COVER_FETCH_WORKERS` 环境变量控制并发数（默认5）

## 2. 统一缓存管理

### 新增缓存管理器
- **文件**: `cache_manager.py`
- **功能**:
  - 单例模式，全局统一管理所有缓存
  - 线程安全的缓存访问
  - 支持缓存统计和清理
  - 懒加载缓存锁，减少内存开销

### 替换分散的缓存
- **修改文件**: `douban.py`, `cover_validator.py`
- **优化**: 将原来分散在各处的缓存字典统一使用缓存管理器
- **好处**:
  - 代码更清晰，易于维护
  - 避免缓存重复定义
  - 便于监控和调试

## 3. 进度条支持

### 添加tqdm进度条
- **文件**: `douban.py`, `cover_validator.py`
- **功能**:
  - 电影同步进度条
  - 图书同步进度条
  - 封面验证进度条
- **效果**: 实时显示处理进度，提升用户体验

## 4. 配置管理优化

### 集中配置管理
- **文件**: `config.py`
- **新增配置**:
  - `MAX_WORKERS`: 最大并发线程数（默认10）
  - `MAX_URL_WORKERS`: URL验证并发数（默认5）
  - `COVER_FETCH_WORKERS`: 封面获取并发数（默认5）

### 环境变量支持
- **文件**: `.env.example`
- **新增环境变量**:
  - `MAX_WORKERS`: 控制整体并发数
  - `URL_VALIDATION_WORKERS`: 控制URL验证并发数
  - `COVER_FETCH_WORKERS`: 控制封面获取并发数

## 5. 依赖更新

### 新增依赖
- **文件**: `requirements.txt`
- **新增**: `tqdm>=4.66.0`（进度条库）

## 性能提升预估

### 封面验证场景（100部电影）
- **优化前**: 串行处理，每个页面验证3个URL，约15分钟
- **优化后**: 并行处理（10线程），约1.5分钟
- **提升**: 约10倍

### 封面获取场景（100本书籍）
- **优化前**: 串行尝试3个源，约25分钟
- **优化后**: 并行尝试，约8分钟
- **提升**: 约3倍

## 使用方法

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量（可选）
在 `.env` 文件中添加：
```bash
# 并发配置
MAX_WORKERS=10
URL_VALIDATION_WORKERS=5
COVER_FETCH_WORKERS=5
```

### 3. 运行同步
```bash
# 同步电影
python -m douban2notion movie

# 同步图书
python -m douban2notion book

# 验证封面
python -m douban2notion.cover_validator all
```

### 4. 查看进度
运行时会自动显示进度条：
```
同步电影: 100%|████████████████████| 100/100 [02:30<00:00, 1.50s/部]
验证电影封面: 100%|████████████████████| 100/100 [01:20<00:00, 1.25个/s]
```

## 注意事项

1. **并发数控制**: 并发数过高可能导致被目标网站限流，建议根据网络环境调整
2. **内存使用**: 缓存会占用内存，长时间运行可定期清理
3. **错误处理**: 并行处理中单个失败不影响整体进度，但会记录日志
4. **兼容性**: 所有优化保持向后兼容，不影响现有功能
